WebDevTrends

Intro

Banyak Teknologi baru seperti Web Component atau PWA ( Progressive Web App) yang sangat berguna bagi
Browser-browser di Tahun 2017.
Artikel kali ini akan membahas mengenai Trend Perkembangan Web di Tahun 2018.

1. Mengerti Dasar

Untuk menjadi Web Developer di tahun 2018 dibutuhkan perhatian khusus pada Fundamental. Yaitu masih dengan
Javacscript,HTML dan CSS. Seorang Web Developer perlu mengetahui bagaimana cara Javascript, Html dan CSS berkerja bersama.

2. Node.Js

Masih sama seperti di tahun 2017 Node.Js masih akan populer di tahun 2017. Mengapa harus Node.js?? karena
Node.Js bisa dikatakan adalah bahasa yang digunakan untuk Backend-Developer dan bahasa dasar yang digunakan adalah Javascript, sehingga memungkinkan untuk menguasai Back-End dan Front-End sekaligus.

3. Lebih Mengenal React, Angular and/ or Vue.js

React, Angular dan Vue.Js adalah framework yang digunakan untuk client-side atau yang juga biasa disebut front-end dengan menggunakan basis bahasa javasript, tetapi react tidak bisa dikatakan sebagai framework, lebih tepatnya adalah library.Ketiga hal di atas menciptakan pengalaman pengguna/user experience yang sangat menarik untuk aplikasi web kamu karena semuanya dikontrol dengan JavaScript. Pengguna mungkin terus menyukai aplikasi web yang menarik, sehingga kerangka kerja frontend seperti Angular, React atau Vue.js akan tetap penting di tahun 2018. Mempelajari salah satu atau semuanya akan menjadi ide yang sangat bagus. Mereka memungkinkan dirimu untuk belajar tentang web developmnet dan menciptakan web yang bagus dengan interface yang interaktif

4. Komponen Web & Kompiler Komponen

Pada akhir 2017, Support Browser masih menjadi masalah yang cukup besar, Tetapi pada awal 2018, Masalah ini berkurang. Empat API Web yang membentuk komponen web yaitu, Unsur Khusus, DOM Shadow, Template HTML dan Impor HTML sudah bisa digunakan oleh Browser-browser ternama yang paling banyak digubakan.

Ide pada web components itu sederhana:Untuk apa menggunakan library seperti React untuk membangun komponen yang hanya berfungsi di library itu sendiri, sedangkan dengan menggunakan vanilla JavaScript dapat membangun komponene web yang juga dapat digunakan luar framework.

5. Penggunaan Static Page Generator

Trend berikutnya di tahun 2018 adalah Penggunaaan Static Page Generator, Static Page Generator adalah aplikasi statis yang memungkinkan kita untuk membuat halaman web pada server lokal atau pun built-in server yang berisikan Html,Css dan Javasript. Keuntungan dari Static Page generator adalah, karena sudah multi platform dan juga dapat menggunakan markdown. Contoh beberapa Generator Static Page adalah Jekyll, Hugo, Nuxt.js dan lain-lain

The Simple Guide Git

Persiapan

  • Unduh Git untuk OSX

  • Unduh Git untuk Windows

  • Unduh Git untuk Linux

Buat Repositori Baru

Dengan cara : git init

Periksa Repositori

Buat salinan kerja dari repositori lokal dengan cara : git clone /jalur/ke/repositori

Saat menggunakan server jarak jauh : git clone namapengguna@host:/jalur/ke/repositori

Alur Kerja

Repositori lokal (trees) dikelolah oleh git. Yang berisi

  • Direktori Kerja

  • Direktori Index

  • Direktori HEAD

    Dan mengarah kepada commit terakhir

Tambah & Commit

Kita dapat melakukan langkah awal perubahan atau penambahan index dengan cara :

git add <namaberkas>

git add

Untuk Commit sepenuh nya kita dapat melakukan dengan cara :

git commit -m "Pesan komit"

Berkas telah bercommit di HEAD, namun belum di repositori dari jarak jauh

Mengirim Perubahan

Saat perubahan telah tersimpan di HEAD salinan kerja lokal, lakukan :

git push origin master

Repositori telah terkirim ke jarak-jauh, dan Ubah master sesuai cabang yang kamu inginkan.

Jika repositori yang ada belum dikloning dan ingin dihubungkan ke server jarak-jauh, kamu perlu menambahkan

git remote add origin <server>

Sekarang kamu bisa mengirimkan perubahan ke server jarak-jauh yang dituju

Cabang

Cabang atau branching digunakan untuk mengembangkan fitur-fitur secara terisolasi. Cabang utama atau master merupakan cabang bawaan ketika kamu membuat repositori. Gunakan cabang lain untuk pengembangan, setelah selesai, gabungkan kembali ke cabang utama.

buat cabang baru dengan nama “fitur_x” dan beralih kedalamnya menggunakan :

git checkout -b fitur_x

beralih kembali ke master

git checkout master

dan hapus cabang yang tadi dibuat

git branch -d fitur_x

suatu cabang tidak terbuka untuk yang lainnya kecuali jika kamu mengirimkannya ke repositori jarak-jauh.

git push origin <cabang>

Perbarui & Gabung

untuk memperbarui repositori lokal ke komit terkini, lakukan :

git pull

dari direktori kerja kamu untuk mengambil dan menggabungkan perubahan jarak-jauh.

untuk menggabungkan cabang lain ke cabang aktif (misal master), gunakan :

git merge <cabang>

pada kasus diatas, git mencoba menggabungkan perubahan secara otomatis. Sayangnya hal ini tak selalu berjalan mulus dan bisa menyebabkan konflik. Kamu lah yang bertanggung jawab menggabungkan konflik tersebut secara manual dengan menyunting berkas yang ditunjukkan git. Setelah itu, kamu perlu memarkahinya dengan :

git add <namaberkas>

sebelum penggabungan berlaku, kamu bisa melakukan analisa menggunakan :

git diff <source_branch> <target_branch>

Menandai

sangat dianjurkan membuat penanda atau tags untuk perangkat lunak yang dirilis. Hal ini amat lah lazim, yang juga terjadi di SVN. Kamu bisa membuat penanda baru dengan nama 1.0.0 dengan menjalankan :

git tag 1.0.0 1b2e1d63ff

1b2e1d63ff adalah 10 karakter pertama dari identitas komit yang ingin kamu referensikan ke penanda.

Log

dalam bentuknya yang paling sederhana, kamu bisa mempelajari riwayat repositori menggunakan :

git log

kamu bisa menambahkan banyak parameter untuk menampilkan log sesuai keinginan. Untuk melihat komit penulis tertentu:

git log --author=bob

Untuk melihat log yang dimampatkan, satu baris per komit:

git log --pretty=oneline

Atau mungkin kamu ingin melihat pohon ASCII art seluruh percabangan disertai nama dan penandanya:

git log --graph --oneline --decorate --all

Sekedar melihat berkas yang berubah:

git log --name-status

Ini baru sedikit saja dari sekian banyak parameter yang bisa kamu gunakan. Lebih jauh lagi, lihat :

git log --help

Mengembalikan perubahan lokal

Seandainya kamu melakukan kesalahan (yang tentunya tak pernah terjadi kamu bisa mengembalikannya menggunakan perintah :

git checkout -- <namaberkas>

perintah di atas mengembalikan perubahan di dalam pokok kerja kamu dengan konten terakhir dari HEAD. Perubahan dan berkas baru yang telah ditambahkan ke indeks akan tetap tersimpan.

Jika kamu ingin menggugurkan perubahan dan komit lokal seutuhnya, ambil riwayat terakhir dari server dan arahkan ke cabang master lokal seperti ini :

git fetch origin

git reset --hard origin/master
